# Kubernetes Training Agent Context

## Purpose

This project contains a structured Kubernetes training program designed for a
Cloud Security Engineer who needs deep operational, architectural, and security
mastery.

## Project Layout

| Path | Purpose |
| --- | --- |
| `README.md` | Program overview, audience, outcomes, and usage |
| `MASTER_CLASS_CURRICULUM.md` | Detailed master-level course outline, labs, delivery schedules, and milestones |
| `labs/module-NN/` | Hands-on lab files for each module: README, manifests, and templates |

## Content Rules

- Keep the material role-focused on Cloud Security Engineering.
- Prefer practical security outcomes over generic Kubernetes theory.
- Cover architecture, operations, detection, prevention, policy, compliance,
  incident response, and platform engineering tradeoffs.
- Use Markdown only unless the user explicitly requests code, manifests, or lab
  assets.
- Expand the curriculum by editing the existing training documents before
  adding new files.

## Definition Of Done

A content update in this project is complete when:

- the training path is clear enough to teach from directly
- the content includes measurable learning outcomes
- hands-on labs or practice expectations are defined
- the material reflects security engineering responsibilities, not only cluster
  administration
